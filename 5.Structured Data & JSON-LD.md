## **სლაიდი 2: რა არის Structured Data?**

> თეორია: სტრუქტურირებული მონაცემები (Structured Data) ეფუძნება Schema.org-ის ვოკაბულარს — Google-ს, Bing-სა და Yandex-ის ერთობლივ სტანდარტს. ის ეხმარება საძიებო სისტემებს გაიგონ თქვენი გვერდის შინაარსი.

Structured Data არის მონაცემთა ფორმატი, რომელიც ეხმარება საძიებო სისტემებს გაიგონ თქვენი გვერდის შინაარსი და აჩვენონ **მდიდარი სნიპეტები** (Rich Snippets) ძიების შედეგებში.

### **რატომ არის მნიშვნელოვანი:**
- 🌟 **Rich Snippets** — ვარსკვლავური რეიტინგები, ფასები, მარაგი პირდაპირ SERP-ში
- 📈 **CTR-ს ზრდა** — მდიდარი შედეგები იზიდავს უფრო მეტ დაწკაპუნებას (35%-მდე)
- 🤖 **Google-სთვის გასაგები** — პროდუქტის თვისებები, არა მხოლოდ ტექსტი

## **სლაიდი 3: რატომ JSON-LD?**

### **რატომ JSON-LD?**

## **სლაიდი 10: React Helmet-async-ით JSON-LD-ის ინექცია**

> თეორია: ლექცია #4-ში ვისწავლეთ, როგორ ჩავსვათ Meta თეგები `react-helmet-async`-ით. იგივე `<Helmet>` კომპონენტი იღებს `<script type="application/ld+json">` თეგსაც — ასე ვინექცირებთ სტრუქტურირებულ მონაცემებს SPA-ში.

### **Reusable Seo კომპონენტი — Meta + JSON-LD ერთად:**

```jsx
// src/components/Seo.jsx
import { Helmet } from 'react-helmet-async';

export default function Seo({ title, description, path, jsonLd }) {
  const siteUrl = 'https://ltdmamu.ge';
  const fullUrl = `${siteUrl}${path}`;

  return (
    <Helmet>
      {/* Meta თეგები (ლექცია #4) */}
      <title>{title} | LTD MAMU</title>
      <meta name="description" content={description} />
      <link rel="canonical" href={fullUrl} />
      <meta property="og:title" content={`${title} | LTD MAMU`} />
      <meta property="og:description" content={description} />

      {/* JSON-LD სტრუქტურირებული მონაცემები (ლექცია #5) */}
      {jsonLd && (
        <script type="application/ld+json">
          {JSON.stringify(jsonLd)}
        </script>
      )}
    </Helmet>
  );
}
```

```jsx
      price: product.price,
      priceCurrency: 'GEL',
      availability: 'https://schema.org/InStock'
    },
    aggregateRating: {
      '@type': 'AggregateRating',
      ratingValue: product.rating,
      reviewCount: product.reviews.length
    }
  };

  return (
    <>
      <Seo
        title={product.name}
        description={product.description}
        path={`/products/${product.category}/${product.slug}`}
        jsonLd={jsonLd}
      />
      <h1>{product.name}</h1>
    </>
  );
}
```

### **Key Points:**
- JSON-LD `JSON.stringify(jsonLd)`-ით გადაიქცევა სტრიქონად `<script>` თეგისთვის
- `jsonLd` პროპსი **არასავალდებულო**ა — გვერდებს, სადაც არ გჭირდება schema, უბრალოდ არ გადასცემთ
- Breadcrumb JSON-LD-ს შეგიძლიათ Breadcrumbs.jsx კომპონენტშიც ჩასვათ იგივე პრინციპით

## **სლაიდი 12: შეჯამება — Structured Data**

### **რა ვისწავლეთ დღეს?**

✅ **რა არის Structured Data** — Schema.org + JSON-LD
✅ **Organization Schema** — ბრენდის ცოდნის პანელი
✅ **Product Schema** — ფასი, მარაგი, რეიტინგი SERP-ში
✅ **Review & Rating Schema** — ვარსკვლავები + სოციალური მტკიცებულება
✅ **Breadcrumb Schema** — ნავიგაციის ჯაჭვი SERP-ში
✅ **Local Business Schema** — Google Maps + ლოკალური SEO
✅ **@graph** — მრავალი Schema ერთ ბლოკში
✅ **React Helmet-async JSON-LD** — სტრუქტურირებული მონაცემების ინექცია SPA-ში

### **ლექცია #4 ↔ #5 კავშირი:**

```
ლექცია #4 (Meta Tags + Helmet)  →  <title>, description, og:, canonical
ლექცია #5 (JSON-LD)            →  Product/Organization/Breadcrumb schema
ორივე ერთად <head>-ში         →  Google-სთვის სრულყოფილი გვერდი
```

### **Key Takeaway:**

> "Meta თეგები Google-ს **სათაურს** აძლევს, JSON-LD კი — **გასაგებ შინაარსს**. ორივე ერთად = Rich Snippets + მაღალი CTR."
