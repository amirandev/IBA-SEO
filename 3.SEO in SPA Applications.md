## სლაიდი 5: 4. წვდომის კონტროლი: Robots.txt

**ტექსტი:**
იმისათვის, რომ რობოტმა სკანირების ბიუჯეტი სწორად დახარჯოს, საიტის „შესასვლელში“ ვახვედრებთ ფაილს **robots.txt** (თავსდება `public/` ფოლდერში).
* **დანიშნულება:** ის მიუთითებს საძიებო სისტემებს, თუ საიტის რომელი ნაწილების დასკანირებაა **აკრძალული** (მაგალითად, ადმინ პანელი ან პირადი მონაცემები).

```text
User-agent: *
Disallow: /admin/
Disallow: /checkout/

Sitemap: https://mysite.com/sitemap.xml
```

**ტექსტი:**
აქ იწყება ფრონტენდ დეველოპერების მთავარი გამოწვევა. React არის SPA (Single Page Application).

## სლაიდი 10: 9. ორფაზიანი ინდექსაცია (Two-Wave Indexing) React-ში

**ტექსტი:**
ამ პრობლემის გადასალახად Google იყენებს სპეციალურ ორფაზიან სისტემას:

1.  **ფაზა 1 (HTML სკანირება):** რობოტი იწერს კოდს. რადგან ტექსტი არ არის, ის გვერდს აგდებს სპეციალურ რენდერინგის რიგში (Render Queue) და მიდის სხვა საიტებზე.
2.  **ფაზა 2 (JS რენდერინგი):** როცა თავისუფალი სერვერული სიმძლავრე გამოუჩნდება (შეიძლება დასჭირდეს საათები ან დღეები), Google რთავს თავის ვირტუალურ ბრაუზერს, უშვებს JavaScript-ს, „ხატავს“ გვერდს და მხოლოდ მაშინ აინდექსებს რეალურ კონტენტს.

## სლაიდი 15: 14. საიტის რუკა (Sitemap.xml) React-ში

**ტექსტი:**
რადგან Google-ს უჭირს React საიტების სკანირება, ჩვენ მას უნდა მივცეთ გზამკვლევი — `Sitemap.xml`.

*   ეს არის XML ფაილი საიტის ყველა აქტიური ლინკის სიით.
*   React პროექტში მას ვათავსებთ `public/` ფოლდერში, რათა ის ხელმისაწვდომი იყოს პირდაპირ მისამართზე: `mysite.com/sitemap.xml`.

```xml
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://mysite.com/</loc>
    <priority>1.0</priority>
  </url>
</urlset>
```

## სლაიდი 17: 16. ერთნაირი სათაურების პრობლემა და React Helmet

**ტექსტი:**
რადგან React-ს აქვს მხოლოდ ერთი `index.html`, საიტის ყველა გვერდს (მთავარს, პროდუქტებს, კონტაქტს) საწყის ეტაპზე ერთი და იგივე სათაური (`<title>`) აქვს, რაც აბნევს საძიებო სისტემას.

**გამოსავალი:** ბიბლიოთეკა `react-helmet-async`. ის საშუალებას გვაძლევს, კონკრეტული გვერდის კომპონენტის შიგნიდან დინამიურად შევცვალოთ ჰედერის თეგები:

# ეტაპი 5 — შექმენით robots.txt

შედით

```
public/
```

და შექმენით ფაილი

```

```

მაგალითი

```txt
User-agent: *

Allow: /

Sitemap: https://YOUR-DOMAIN/sitemap.xml
```

### რას ნიშნავს?

**User-agent**

რომელი საძიებო სისტემა კითხულობს ამ ფაილს.

```
*
```

ნიშნავს ყველას.

---

**Allow: /**

საძიებო სისტემას შეუძლია საიტის ყველა გვერდის ინდექსაცია.

---

**Sitemap**

უთითებს სად მდებარეობს საიტის Sitemap.

# ეტაპი 6 — შექმენით sitemap.xml

ასევე public საქაღალდეში.

```
public/

sitemap.xml
```

მაგალითი

```xml
<urlset>

    <url>
        <loc>https://example.com/</loc>
    </url>

    <url>
        <loc>https://example.com/about</loc>
    </url>

    <url>
        <loc>https://example.com/contact</loc>
    </url>

</urlset>
```

---

### რას ნიშნავს?

**url**

ერთი გვერდის აღწერაა.

**loc**

გვერდის მისამართი.

Sitemap-ის დახმარებით Google უფრო სწრაფად პოულობს ყველა გვერდს.
