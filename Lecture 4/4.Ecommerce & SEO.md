## **სლაიდი 3: HTML ბმულები & SEO**

> თეორია: ბმულები არის ინტერნეტის საფუძველი. Google-ის PageRank ალგორითმი აფასებს გვერდის ავტორიტეტს მასზე მიმთითებელი ბმულების რაოდენობითა და ხარისხით. სემანტიკური ბმულები ეხმარება საძიებო სისტემებს კონტექსტის გაგებაში.

### **რატომ არის ბმულები მნიშვნელოვანი:**

ბმულები არის საფუძველი:
- **ნავიგაცია** (მომხმარებლის გამოცდილება)
- **დათვალიერებადობა** (საძიებო სისტემები პოულობენ თქვენს გვერდებს)
- **გვერდის ავტორიტეტი** (ლინკ წვენის განაწილება)

### **საუკეთესო პრაქტიკა ბმულებისთვის:**

```html
<!-- ✅ GOOD: Descriptive, semantic links -->
<a href="/products/wireless-bluetooth-headphones" 
   title="Premium Wireless Bluetooth Headphones with Noise Cancelling">
   Wireless Bluetooth Headphones
</a>

<!-- ❌ BAD: Generic, non-descriptive -->
<a href="/product.php?id=123">Click here</a>
```

### **ძირითადი ბმულის ატრიბუტები:**

1. **`href`** - დანიშნულების URL (გამოიყენეთ სუფთა, წაკითხვადი მისამართები)
2. **`title`** - ტულტიპის ტექსტი (აწვდის კონტექსტს)
3. **`rel`** - ურთიერთობა (nofollow, noopener, canonical, alternate)
4. **`target`** - სად გაიხსნას (_blank, _self)

> 📌 React-ში ბმულები ყოველთვის `<Link to="...">` კომპონენტით უნდა იქმნას (არა `<button>`), რათა Googlebot-მა ნამდვილი `<a href>` იხილოს. ეს დეტალურად განხილულია ლექცია #3-ში და ქლასვორკში.

## **სლაიდი 4: Title ატრიბუტის დეტალური მიმოხილვა**

> თეორია: title ატრიბუტი წარმოადგენს დამატებით სემანტიკულ ფენას. მიუხედავად იმისა, რომ Google-მა შეამცირა მისი მნიშვნელობა რანჟირებაში, ის მაინც მნიშვნელოვანია ხელმისაწვდომობისა და მომხმარებლის გამოცდილებისთვის.

### **რა არის Title ატრიბუტი?**

`title` ატრიბუტი გვაწვდის **დამატებით ინფორმაციას** ელემენტის შესახებ, ნაჩვენებია ტულტიპის სახით ჰოვერის დროს.

### **გავლენა SEO-ზე:**
- ✅ აუმჯობესებს **ხელმისაწვდომობას**
- ✅ აწვდის **კონტექსტს** მომხმარებლებს
- ✅ ეხმარება საძიებო სისტემებს **ბმულის რელევანტურობის** გაგებაში
- ⚠️ გამოიყენეთ ზომიერად - ნუ გადატვირთავთ საკვანძო სიტყვებით!

### **მაგალითები:**

```html
<!-- Product Link with Title -->
<a href="/products/steel-pipe-fitting" 
   title="ფოლადის მილტუჩი - მაღალი ხარისხის, 5 წლიანი გარანტია">
   Steel Pipe Fitting

<!-- Category Link -->
<a href="/category/water-supply" 
   title="წყალმომარაგების სისტემები და აქსესუარები">
   Water Supply Systems
</a>

<!-- Navigation -->
<nav>
  <a href="/" title="მთავარი გვერდი - LTD MAMU">Home</a>
  <a href="/catalog" title="პროდუქციის კატალოგი">Catalog</a>

```

### **საუკეთესო პრაქტიკა:**
- შეინახეთ **მოკლედ** (50-60 სიმბოლო)
- გახადეთ **აღწერილობითი** და სასარგებლო
- ჩართეთ **საკვანძო სიტყვები ბუნებრივად**

## **სლაიდი 5: Alt ატრიბუტი სურათებისთვის**

> თეორია: საძიებო სისტემები ვერ "ხედავენ" სურათებს — ისინი კითხულობენ alt ტექსტს. alt ტექსტი უზრუნველყოფს ხელმისაწვდომობას და ეხმარება Google-ს სურათების ინდექსირებაში, რაც მნიშვნელოვანია ვიზუალური ძიებისთვის.

### **რატომ არის Alt ტექსტი კრიტიკული ელექტრონული კომერციისთვის:**

1. **ხელმისაწვდომობა** - ეკრანის წამკითხველები აღწერენ სურათებს მხედველობის შეზღუდული მქონე მომხმარებლებისთვის
2. **SEO** - Google-ს არ შეუძლია სურათების "დანახვა", ის კითხულობს alt ტექსტს
3. **სათადარიგო** - გამოჩნდება, როდესაც სურათები არ იტვირთება
4. **სურათების ძიება** - ჩნდება Google-ის სურათების ძიებაში

### **Alt ტექსტის სწორი განხორციელება:**

```html
<!-- ✅ EXCELLENT: Descriptive, includes product details -->
<img src="steel-pipe-fitting.jpg" 
     alt="ფოლადის მილტუჩი MAMU ბრენდი, მოდელი 925872, SKU: 446310786"
     width="500" 
     height="500">

<!-- ✅ GOOD: Clear and descriptive -->
<img src="headphones.jpg" 
     alt="Wireless Bluetooth Headphones - Noise Cancelling, Black">

<!-- ❌ BAD: Too generic -->
<img src="product.jpg" alt="Product">

<!-- ❌ BAD: Keyword stuffing -->
<img src="pipe.jpg" alt="pipe fitting steel pipe best price buy pipe fitting">

<!-- ✅ DECORATIVE: Empty alt (screen readers skip it) -->
<img src="decorative-border.png" alt="">
```

### **ელექტრონული კომერციის სურათების Alt ტექსტის ფორმულა:**

```
[Product Name] + [Brand] + [Key Feature/Color/Size] + [Model/SKU if relevant]
```

**მაგალითი:**
```html
<img src="laptop.jpg" 
     alt="MacBook Pro 16-inch, Apple M2 Chip, 32GB RAM, Space Gray">
```

### **გავლენა SEO-ზე:**
- სურათები სწორი alt ტექსტით რეიტინგდებიან **უფრო მაღლა სურათების ძიებაში**
- ხელს უწყობს **გვერდის საერთო რელევანტურობას**
- აუმჯობესებს **ხელმისაწვდომობის ქულას** (WCAG-სთან შესაბამისობა)

## **სლაიდი 6: ელექტრონული კომერციის გვერდის ბმულების სტრუქტურა**

> თეორია: URL სტრუქტურა არის საიტის იერარქიის ანარეკლი. Google იყენებს URL-ებს გვერდების კონტექსტის გასაგებად. ლოგიკური, იერარქიული URL სტრუქტურა ეხმარება crawler-ებს ეფექტურად დაათვალიერონ საიტი.

### **იერარქიული URL სტრუქტურა:**

```
Homepage
├── Category Pages
│   ├── Subcategory Pages
│   │   ├── Product Pages
│   │   └── Product Pages
│   └── Subcategory Pages
└── Information Pages (About, Contact, Blog)
```

### **URL-ის საუკეთესო პრაქტიკა:**

```html
<!-- ✅ CLEAN, SEO-FRIENDLY URLs -->
https://ltdmamu.ge/water-supply/pipe-fittings/steel-pipe-fitting-925872

<!-- ❌ DYNAMIC, NON-DESCRIPTIVE URLs -->
https://ltdmamu.ge/product.php?id=925872&cat=45

<!-- Breadcrumb Navigation (HTML) -->
<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">Home</a></li>
    <li><a href="/water-supply">Water Supply</a></li>
    <li><a href="/water-supply/pipe-fittings">Pipe Fittings</a></li>
    <li><span>Steel Pipe Fitting</span></li>
  </ol>
</nav>
```

> 📌 Breadcrumb-ის **HTML ნავიგაცია** რჩება აქ (SEO-სთვის მნიშვნელოვანია). Breadcrumb-ის **JSON-LD სტრუქტურირებული მონაცემები** გადატანილია **ლექცია #5-ში**.

### **ძირითადი პრინციპები:**
1. **ბრტყელი არქიტექტურა** - მაქსიმუმ 3 დაწკაპუნება ნებისმიერ პროდუქტამდე
2. **აღწერილობითი URL-ები** - ჩართეთ საკვანძო სიტყვები
3. **Breadcrumbs** - ეხმარება მომხმარებლებსა და საძიებო სისტემებს ნავიგაციაში
4. **შიდა ბმულები** - ანაწილებს გვერდის ავტორიტეტს
5. **Sitemap** - XML sitemap დათვალიერების რობოტებისთვის (ნახსენებია ქლასვორკში)

## **სლაიდი 7: International & hreflang SEO**

> თეორია: Google-ის მიზანია აჩვენოს მომხმარებლის ენასა და რეგიონთან შესაბამისი შედეგები. hreflang ტეგები არის სიგნალი, რომელიც ეხმარება Google-ს აირჩიოს სწორი ენობრივი ვერსია.

### **hreflang-ის იმპლემენტაცია:**

```html
<!-- ✅ ენის მიზნობრიობა ქართულისა და ინგლისურისთვის -->
<link rel="alternate" hreflang="ka" href="https://ltdmamu.ge/ka/product/steel-pipe-fitting" />
<link rel="alternate" hreflang="en" href="https://ltdmamu.ge/en/product/steel-pipe-fitting" />
<link rel="alternate" hreflang="x-default" href="https://ltdmamu.ge/ka/product/steel-pipe-fitting" />
```

### **hreflang-ის საუკეთესო პრაქტიკა:**

- ✅ გამოიყენეთ **სრული აბსოლუტური URL-ები** (არა ფარდობითი ბილიკები)
- ✅ ჩართეთ **თვითრეფერენსული** hreflang (გვერდი საკუთარ თავზე მიუთითებს)
- ✅ გამოიყენეთ **x-default** როგორც სათადარიგო (ჩვეულებრივ, ძირითადი ენა)
- ✅ გამოიყენეთ **ISO 639-1** ენის კოდები (`ka`, `en`, `de`)
- ⚠️ არასოდეს აწინააღმდეგოთ hreflang-ისა და canonical-ის ტეგები

### **ენის ამომრჩევლის UX:**

```html
<nav class="language-switcher" aria-label="Language selector">
  <a href="/ka/product" hreflang="ka" lang="ka" rel="alternate">ქართული</a>
  <a href="/en/product" hreflang="en" lang="en" rel="alternate">English</a>
</nav>
```

## **სლაიდი 8: Meta თეგები — SEO-ს ფუნდამენტი**

> თეორია: `<head>`-ში მოთავსებული Meta თეგები არის პირველი (და ხშირად ერთადერთი) ინფორმაცია, რასაც Googlebot ხედავს საწყის HTML-ში. სწორად მართული Meta თეგები განსაზღვრავს, თუ როგორ გამოჩნდება თქვენი გვერდი SERP-ში, სოციალურ ქსელებში და საძიებო სისტემებში.

### **ძირითადი Meta თეგები — რატომ გვჭირდება თითოეული?**

```html
<!-- 1. TITLE — ყველაზე მნიშვნელოვანი SEO თეგი -->
<!-- Google SERP-ში ეს არის ლურჯი სათაური. მომხმარებელი ამით წყვეტს დაგაკლიკოს თუ არა -->
<title>ფოლადის მილტუჩი MAMU 925872 | LTD MAMU</title>

<!-- 2. META DESCRIPTION — SERP-ში ნაჩვენები ტექსტი. ზრდის CTR-ს -->
<meta name="description" content="მაღალი ხარისხის ფოლადის მილტუჩი MAMU. SKU: 446310786. 25.99₾. უფასო მიწოდება მთელს საქართველოში." />
```

## **სლაიდი 9: React Helmet-async — დინამიური Meta თეგები SPA-ში**

> თეორია: React SPA-ს აქვს **ერთი** `index.html`, რომელშიც მხოლოდ ერთი `<title>` და `<meta>` თეგია. როცა მომხმარებელი გადადის გვერდებზე, ბრაუზერი თავიდან არ იტვირთება, ამიტომ სათაური და მეტა თეგები არ შეიცვლება. `react-helmet-async` საშუალებას გვძლევს, **თითოეულ გვერდის კომპონენტიდან** დინამიურად ჩავაგდოთ სწორი `<title>`, `<meta>` და `<link>` თეგები `<head>`-ში.

### **რატომ გვჭირდება Helmet?**

```
❌ ცუდი (SPA-ის ნაგავი):
   index.html → <title>React App</title>   (ყველა გვერდს ერთი და იგივე სათაური აქვს!)

✅ კარგი (Helmet-ით):
   /products/steel-pipe-fitting → <title>ფოლადის მილტუჩი | LTD MAMU</title>
   /about                       → <title>ჩვენ შესახებ | LTD MAMU</title>
   /contact                     → <title>კონტაქტი | LTD MAMU</title>
```

### **ნაბიჯი 4: Reusable Seo კომპონენტი (რეკომენდირებული)**

ყველა გვერდზე ერთი და იგივე Meta თეგების გასაწერად, შევქმნათ გამრეცხავი `Seo.jsx`:

```jsx
// src/components/Seo.jsx
import { Helmet } from 'react-helmet-async';

export default function Seo({ title, description, keywords, path, image, type }) {
  const siteUrl = 'https://ltdmamu.ge';
  const siteName = 'LTD MAMU';
  const fullUrl = `${siteUrl}${path}`;
  const fullTitle = keywords ? `${title} | ${siteName}` : `${title} | ${siteName}`;

  return (
    <Helmet>
      <title>{title} | {siteName}</title>
      <meta name="description" content={description} />
      {keywords && <meta name="keywords" content={keywords} />}
      <meta name="robots" content="index, follow" />
      <link rel="canonical" href={fullUrl} />
      <meta property="og:title" content={`${title} | ${siteName}`} />
      <meta property="og:description" content={description} />
      <meta property="og:url" content={fullUrl} />
      <meta property="og:type" content={type || 'website'} />
      {image && <meta property="og:image" content={image} />}
      <meta name="twitter:card" content="summary_large_image" />
      <meta name="twitter:title" content={`${title} | ${siteName}`} />
      <meta name="twitter:description" content={description} />
      {image && <meta name="twitter:image" content={image} />}
    </Helmet>
  );
}

// გამოყენება:
// <Seo title="მთავარი" description="..." path="/" />
// <Seo title="ფოლადის მილტუჩი" description="..." path="/products/water-supply/steel-pipe-fitting" image="..." type="product" />
```

### **ნაბიჯი 5: დინამიური Meta თეგები როუთიდან (useLocation / useParams)**

React Router-ის ჰუკებით ვიღებთ მიმდინარე URL-სა და პარამეტრებს, რათა Meta თეგები **ავტომატურად** შეივსოს:

```jsx
import { Helmet } from 'react-helmet-async';
import { useParams, useLocation } from 'react-router-dom';

export default function ProductDetail() {
  const { category, productId } = useParams();
  const location = useLocation();

  const productName = productId.replace(/-/g, ' ').replace(/\b\w/g, c => c.toUpperCase());
  const fullUrl = `https://ltdmamu.ge${location.pathname}`;

  return (
    <Helmet>
      <title>{productName} | LTD MAMU</title>
      <meta name="description" content={`იყიდე ${productName} ონლაინ. საუკეთესო ფასი საქართველოში.`} />
      <link rel="canonical" href={fullUrl} />
      <meta property="og:url" content={fullUrl} />
    </Helmet>
  );
}
```

**Key Points:**
- `useLocation()` → აძლევს მიმდინარე URL-ს (pathname + search) — canonical/og:url-ისთვის
- `useParams()` → აძლევს URL-ში ჩადგმულ პარამეტრებს (category, productId)
- `useSearchParams()` → აძლევს query პარამეტრებს (?sort=price&page=2)
- Canonical და og:url თეგებში ყოველთვის **სრული URL** უნდა იყოს (`https://domain.com/path`)

> 📌 **JSON-LD სტრუქტურირებული მონაცემების** Helmet-ით ჩასმა (Product/Organization/Breadcrumb Schema) გადატანილია **ლექცია #5-ში**.

## **სლაიდი 11: Core Web Vitals (ძირითადი ვებ-ვიტალები)**

> თეორია: Core Web Vitals ეფუძნება რეალური მომხმარებლის გამოცდილების გაზომვას (RUM). Google იყენებს Chrome-ის მონაცემებს (CrUX) რეალური მომხმარებლების ჩატვირთვის, ინტერაქტიულობისა და ვიზუალური სტაბილურობის შესაფასებლად.

### **რა არის Core Web Vitals?**

| მეტრიკა | რას ზომავს | კარგი ქულა | გავლენა ელექტრონულ კომერციაზე |

## **სლაიდი 13: შეჯამება — Meta Tags & React Helmet**

### **რა ვისწავლეთ დღეს?**
